# PDF Report Data Mapping

Status: Draft  
Owner: PMG Tracker 360  
Created: 2026-06-25

## 1. Purpose

This document maps each planned PDF report field to the expected source data in the PMG Tracker 360 database/server layer.

The objective is to remove guesswork before implementation. Developers should know which server query or database field feeds each report column, summary card, and metadata value.

## 2. Mapping Rules

- Data fetching must happen in the app/server layer, not inside the generic PDF renderer.
- The shared PDF engine should receive report-ready data only.
- All tracker report queries must be scoped by `organizationId`.
- Soft-deleted records should be excluded unless a specific report requires archived/deleted records.
- Currency fields should be formatted in the PDF renderer or report formatter, not directly in raw SQL.
- Date fields should be normalised to a display format before rendering.
- Optional fields must display `-` or `Not captured` instead of crashing the PDF.

## 3. Common Report Metadata

| Report Field | Source | Notes |
|---|---|---|
| Report title | Static report config | Example: `Tender Register Report` |
| Generated date | Server date/time | Use South African date formatting if applicable |
| Organization name | `organization.name` | Required for tracker reports |
| Organization ID | Active organization/session context | Used for access control and filtering |
| Generated by | Session user | Optional in MVP |
| Selected filters | API query params | Convert values into labels |
| Page number | PDF engine | Generated during PDF rendering |

- `client`

## 9. Formatting Rules

### Dates

Use a consistent display format, for example:

```txt
25 Jun 2026
```

### Currency

Use South African Rand formatting:

```txt
R 12,345.67
```

### Status Labels

Convert snake_case or lowercase values into readable labels.

Examples:

```txt
under_preparation -> Under Preparation
purchase_ordered -> Purchase Ordered
partially_delivered -> Partially Delivered
```

### Empty Values

Use:

```txt
-
```

for missing optional table values.

For important missing business values, use:

```txt
Not captured
```

## 10. Implementation Notes

- Confirm exact schema names before coding fields marked `Confirm exact schema`.
- Prefer existing server functions where they already enforce authorization.
- Avoid duplicating heavy database logic inside PDF builders.
- For MVP, avoid chart rendering unless simple summary tables are enough.
- Keep data mapping functions testable and independent of PDF rendering.
